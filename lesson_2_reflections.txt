What happens when you initialize a repository? Why do you need to do it?
It creates a git repository in your directory, so that you can track your commit history. 

How is the staging area different from the working directory and the repository? What value do you think it offers?
Staging area allows you to choose which files should be commited together in the repository. 

How can you use the staging area to make sure you have one commit per logical change?
You can compare the differences between files in the working directory, staging area, and repository, before making any commits. You can also add multiple files to staging area to ensure that relvant changes in multiple files are committed together as one commit rather than mutliple commits.

What are some situations when branches would be helpful in keeping your history organized? How would branches help?
Branches will allow you to experiment, as well as create new versions without disrupting the master branch codes. Since master branch will be different from other branches, it will be easier to switch across and modify.
